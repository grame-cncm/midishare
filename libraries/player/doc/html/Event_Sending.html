Event Senders

 

The event sender is responsible to handle events to be played. This component takes care of the current track state (solo, mute) before sending events. All event senders must implement the TEventSenderInterface. These are currently two event senders available :

  • TMidiPlayer : a event sender which sends Midi event normally.
  • TQuickTimePlayer : a event sender which play events on QuickTime sound channels.
