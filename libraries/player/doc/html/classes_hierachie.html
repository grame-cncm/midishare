Behavioral Patterns

Behavioral patterns encapsulate algorithms and the assignement of responibilities between objects by defining the communication between them as separate objects.

- An Iterator is a structure that is used to traverse composites. An iterator provides a way to sequentially access elements of an aggregate object without knowing the internal objet representation or collection structure.

For the score management; the class TScoreIterator allows to traverse the score. Moreover the TScoreIterator is a robust iterator. It means that elements can be inserted and removed in the score while iterators are defined on the score.

- A visitor allows to add news operations on the elements of the composite structure without having to modify les classes used in the composite structure.

For the score management; the class TScoreVisitor allows to add operations on the score elemnts.

 

The Chain of Responsability patterns avoid coupling the sender of a request to its receiver by giving more than one object a chance to handle the request. Chain the receiving objects and pass the request along the chain until an object handles it.
